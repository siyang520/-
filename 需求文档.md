# 李思阳 个人作品集单页HTML网站需求文档

## 一、项目概述

### 1.1 项目定位

本项目为李思阳专属个人品牌静态单页HTML网站，作为线上官方名片与能力展示载体。
网站主要用于求职背书、商务对接、作品沉淀和个人品牌展示，面向招聘方、商务合作伙伴、同行创作者统一开放。

### 1.2 核心个人信息

- 姓名：李思阳
- 身份：网络与新媒体专业在校生、短视频内容创作者、短视频剪辑师
- 核心实绩：个人抖音账号单条视频300W+播放，单月从零起号涨粉2000+；NCDA数字艺术大赛湖北省二等奖；熟练掌握PR/AE/PS全流程剪辑
- 核心方向：短视频内容全链路生产、账号运营

### 1.3 目标受众

1. 企业HR、MCN机构：核验内容创作、剪辑、运营能力，作为实习/校招求职的能力背书
2. 品牌方、商务合作方：承接短视频剪辑、账号代运营、内容策划、轻量网页工具定制合作
3. 同行创作者：交流内容创作、流量运营
4. 个人沉淀：统一对外作品入口，集中归档个人项目与创作成果

## 二、视觉设计规范（包豪斯风格）

### 2.1 核心设计原则

- 几何构图：以直线、矩形、方块为基础元素分割页面，构建清晰骨架
- 粗黑边框：所有卡片、图片、模块容器统一使用厚重纯黑外边框，强化视觉边界
- 三原色色块：以正红、正黄、正蓝小面积色块作为点缀与功能区分，不做大面积铺色
- 硬阴影效果：全局采用直角无模糊硬投影，强化硬朗工业质感
- 非对称布局：板块与卡片采用错落非对称排布，规避呆板对称网格
- 大字标题：无衬线粗体大字号标题，形成强视觉层级
- 大面积留白：保证页面呼吸感，信息层级清晰分明

### 2.2 色彩规范

- 基底色：纯黑 #000000、纯白 #FFFFFF
- 点缀三原色：正红 #E6212A、正黄 #FFD100、正蓝 #0052D9
- 阴影色：纯黑硬阴影，无羽化模糊效果

### 2.3 字体规范

- 全局采用无衬线字体体系，保证现代感与可读性
- 页面主标题：超大字号、超粗字重，强化首屏视觉冲击力
- 板块标题：大字号粗体，作为区块视觉锚点
- 正文内容：常规字重，行高适中，保证长文本阅读舒适度
- 数据数字：采用等宽字体，突出量化实绩信息

## 三、页面模块结构（单页纵向滚动）

页面自上而下分为五个核心模块，支持单页滚动与锚点平滑跳转。

### 3.1 Hero 首屏模块

- 核心主标题：李思阳
- 副标题定位：短视频剪辑 · 新媒体运营
- 个人 Slogan：以内容承接流量，以技术落地创意
- 装饰元素：搭配几何色块、粗边框几何图形做包豪斯风格点缀
- 功能按钮：设置「查看作品」「联系我」两个锚点按钮，样式为粗黑边框+三原色填充

### 3.2 关于我 & 核心实绩模块

采用左右非对称布局，左侧为个人简介，右侧为数据卡片：

- 个人简介：介绍专业背景、核心技能矩阵、创作方向与开发能力边界
- 核心实绩卡片（三原色卡片区分）：
  1. 单条视频最高播放 300W+
  2. 单月从零起号涨粉 2000+
  3. NCDA数字艺术大赛 湖北省二等奖
  4. 全流程剪辑

### 3.3 核心能力优势模块

以粗边框卡片形式呈现四大核心优势，便于受众快速感知价值：

1. 全链路内容生产：可独立完成脚本策划、拍摄执行、后期剪辑、封面设计、数据复盘全流程
2. 平台流量运营认知：具备从0到1起号实战经验，熟悉短视频平台算法逻辑与完播互动优化方法
3. 团队协作统筹能力：有项目带队经验，可把控进度、协调分工、稳定交付内容成果
4. 剪辑节奏与视觉表达：强化内容传播效果的节奏与视觉把控

### 3.4 短视频代表作品模块

- 板块大标题：短视频作品集

单张作品卡片必填字段：
- 视频封面图（带粗黑边框+硬阴影）
- 作品标题
- 核心数据（播放量/涨粉量/获奖荣誉等量化信息）
- 内容简介（1-2句话概括视频核心内容）
- 能力标签（如剪辑节奏、分镜设计、文案策划、流量优化等）

布局规则：非对称错落网格排布，卡片 hover 时轻微上浮、阴影加重，保持极简交互逻辑。

### 3.5 联系与页脚模块

- 板块标题：合作联系
- 联系方式：邮箱、合作微信（可选）
- 附加入口：个人简历 PDF 下载按钮
- 合作说明：承接实习对接、短视频剪辑合作、账号运营、轻量网页工具定制等业务
- 底部装饰：简约几何线条与色块装饰，保持包豪斯风格统一性

## 四、交互与响应式要求

### 4.1 交互动效

- 全局滚动平滑过渡，导航锚点点击后平滑跳转至对应区块
- 卡片 hover：轻微向上位移，硬阴影范围加大，无多余花哨动效
- 按钮 hover：底色填充切换，边框保持粗黑样式，反馈清晰克制
- 整体动效简洁克制，服务于功能体验，不破坏包豪斯硬朗风格

### 4.2 响应式适配

- 适配桌面端、平板端、手机端三端显示
- 移动端自动调整为单列布局，保留粗边框、三原色核心视觉元素
- 文字大小与间距自适应，保证移动端阅读清晰、操作顺手

## 五、技术实现要求

1. 单文件 HTML 实现，CSS 与 JS 内嵌，打开即可使用，无需额外依赖
2. 代码结构清晰，关键区块添加注释，便于后续自行修改作品内容与个人信息
3. 图片采用本地相对路径引用，支持离线正常显示
4. 所有外部跳转链接预留占位符，可直接替换为真实地址
5. 采用原生 HTML+CSS+JavaScript 实现，不引入复杂框架，保证页面加载速度
